## 1. 项目概述

### 1.1 项目背景

当前大多数 AI 聊天产品仍然停留在“文本窗口”或“语音助手”阶段，用户与 AI 的交互主要依赖输入框、语音按钮和消息气泡。这种交互形式功能强，但缺少身体感、陪伴感和情绪反馈。

本项目希望利用智能手机已有的硬件能力，将手机转化为一个具备“看、听、说、表情、触碰反馈和运动感知”的小型具身情感聊天机器人。

在这个设计中：

- 手机屏幕是机器人的脸；
- 前置摄像头是机器人的眼睛；
- 麦克风是机器人的耳朵；
- 扬声器是机器人的嘴巴；
- 触屏是机器人的皮肤；
- 陀螺仪和加速度计是机器人的身体感知；
- 震动马达是机器人的触觉反馈；
- LLM、多模态模型和情绪引擎是机器人的大脑。

### 1.2 产品定义

本产品是一款运行在手机上的具身情感聊天机器人 App。

用户可以将手机放在桌面、床头、车内或随身携带。机器人通过屏幕显示极简眼睛和嘴巴，通过摄像头感知用户，通过麦克风听取语音，通过扬声器进行语音回复，并根据用户触摸、摇晃、拿起、靠近、翻转等行为做出拟人化反应。

产品核心不是单纯问答，而是建立一种“手机里的 AI 伙伴”体验。

### 1.3 一句话定位

把手机变成一个会看、会听、会说、会眨眼、会被触摸回应的 AI 情感伙伴。

### 1.4 核心价值

1. 降低 AI 交互门槛  
   用户不需要打开聊天窗口输入文字，可以像和一个小伙伴说话一样自然交流。

2. 增强陪伴感  
   通过表情、声音、触摸、震动、动作反应形成拟生命感。

3. 复用手机硬件  
   不需要额外购买机器人硬件，旧手机也可以变成桌面 AI 伙伴。

4. 支持本地模型和云端模型混合  
   可根据用户隐私、成本和模型能力需求灵活部署。

5. 可扩展为学习助手、工作助手、老人陪伴、儿童陪伴、桌面宠物等多种场景。

## 2. 产品目标

### 2.1 MVP 目标

MVP 阶段目标是验证“手机具身 AI 伙伴”的核心体验是否成立。

MVP 需要实现：

- 手机屏幕显示简化眼睛和嘴巴；
- 用户可以语音唤醒或点击唤醒；
- 机器人可以听懂语音并进行语音回复；
- 回复时屏幕嘴巴同步动效；
- 机器人具备基础情绪表情；
- 用户点击、长按、摇晃、拿起、倒扣手机时机器人有对应反应；
- 机器人可以根据低电量、充电、夜间等设备状态表现不同情绪；
- 支持基础人格 Prompt；
- 支持结构化 AI 输出控制表情、语音和震动；
- 支持桌面常亮模式。

### 2.2 中期目标

中期目标是增强机器人的感知能力和陪伴能力。

需要实现：

- 前置摄像头人脸检测；
- 用户是否在场检测；
- 用户距离判断；
- 基础表情识别；
- 简单记忆系统；
- 主动提醒；
- 专注陪伴模式；
- 夜间低打扰模式；
- 本地 AI Gateway；
- 本地模型与云端模型自动路由。

### 2.3 长期目标

长期目标是形成一个可持续成长的手机具身机器人平台。

包括：

- 多人格系统；
- 长期记忆与关系成长；
- 多模态视觉问答；
- 物体识别；
- 日程、邮件、任务、智能家居工具调用；
- 儿童安全模式；
- 老人关怀模式；
- 跨设备同步；
- App Store / Android 应用市场正式上线；
- 后续可扩展到平板、桌面屏、车载屏和智能音箱。

## 3. 目标用户

### 3.1 核心用户

1. AI 爱好者  
   喜欢尝试新型 AI 交互方式，希望把旧手机变成 AI 桌面机器人。

2. 独居年轻人  
   希望获得轻量陪伴、情绪回应、日常聊天和生活提醒。

3. 学生和办公人群  
   希望获得学习陪伴、专注提醒、任务拆解和情绪鼓励。

4. 家庭用户  
   希望给儿童或老人提供一个低成本的语音互动陪伴设备。

5. 开发者和极客  
   希望本地部署 LLM，并通过手机获得更生动的交互终端。

### 3.2 典型场景

#### 场景一：桌面 AI 伙伴

用户将手机横放在电脑旁。屏幕上显示一双眼睛。用户靠近时，眼睛看向用户；用户说话时，机器人进入倾听状态；回复时嘴巴动起来，并通过语音回答。

#### 场景二：工作陪伴

用户工作 45 分钟后，机器人轻声提醒：

> 你已经专注很久了，要不要起来活动一下？我可以等你回来。

#### 场景三：情绪陪伴

用户说：

> 今天有点累。

机器人判断用户情绪偏低，切换成关心表情，用温柔语气回复：

> 听起来你今天消耗挺大。要不要我陪你简单复盘一下，还是先安静陪你一会儿？

#### 场景四：宠物式互动

用户摇晃手机，机器人眼睛变成晕圈，并说：

> 哇，别晃啦，我有点头晕。

用户给手机充电，机器人闭眼微笑：

> 补充能量中，我感觉好多啦。

## 4. 产品功能范围

### 4.1 MVP 功能清单

| 模块 | 功能 | 优先级 |
|---|---|---|
| 表情系统 | 眼睛、嘴巴、眨眼、嘴型动画 | P0 |
| 语音输入 | 麦克风录音、语音识别 | P0 |
| 语音输出 | TTS 播放、语音状态回调 | P0 |
| 对话系统 | LLM 对话、人格 Prompt | P0 |
| 情绪系统 | 开心、思考、困惑、关心、困倦、低电量、充电 | P0 |
| 触摸交互 | 点击、长按、双击屏幕 | P0 |
| 运动感知 | 拿起、摇晃、倒扣、横竖屏识别 | P0 |
| 设备状态 | 电量、充电、时间、亮度状态 | P0 |
| 结构化输出 | AI 返回 text、emotion、expression、voice、haptic | P0 |
| 设置页 | 人格、语音、摄像头开关、隐私模式 | P1 |
| 桌面模式 | 常亮、低功耗、横屏展示 | P1 |
| 简单记忆 | 用户偏好、名字、长期设置 | P1 |
| 摄像头感知 | 人脸存在检测 | P2 |

### 4.2 非 MVP 功能

以下功能不建议第一版实现：

- 复杂 3D 数字人；
- 高精度唇形同步；
- 多人脸识别；
- 高级情绪识别；
- 完整长期记忆图谱；
- 儿童内容审核系统；
- 完整任务代理系统；
- App 内社交关系；
- 复杂机器人动作系统。
