## 17. 工作模式设计

### 17.1 普通聊天模式

用户主动说话，机器人回复。

### 17.2 桌面陪伴模式

手机常亮显示表情，减少主动打扰，只在用户触发或长时间状态变化时回应。

### 17.3 专注模式

机器人减少主动发言，只做轻量提醒。

适用：

- 学习；
- 编程；
- 写作；
- 会议前准备。

### 17.4 夜间模式

特点：

- 降低屏幕亮度；
- 降低语音音量；
- 关闭主动提醒；
- 表情进入困倦状态；
- 可通过轻触唤醒。

### 17.5 隐私模式

特点：

- 关闭摄像头；
- 关闭持续麦克风监听；
- 不保存长期记忆；
- 只允许点击后对话；
- 本地处理优先。

## 18. 主动行为设计

### 18.1 主动发言原则

机器人可以主动发言，但必须克制。

原则：

1. 不频繁打扰；
2. 用户工作时少说话；
3. 夜间默认不主动说话；
4. 主动发言必须有明确理由；
5. 用户可关闭主动发言；
6. 用户多次忽略后自动降低频率。

### 18.2 可主动发言的场景

| 场景 | 示例 |
|---|---|
| 长时间专注 | 要不要休息一下？ |
| 用户回到画面 | 你回来啦。 |
| 电量低 | 我快没电了，要不要充电？ |
| 充电完成 | 能量补满啦。 |
| 早晨首次见面 | 早上好，今天想先做什么？ |
| 用户语气低落 | 我感觉你有点累，要不要聊聊？ |

### 18.3 主动发言频率

建议提供三档：

- 安静：几乎不主动发言；
- 平衡：重要状态变化时提醒；
- 活泼：更像数字宠物，但仍限制频率。

## 19. 安全与隐私设计

### 19.1 权限原则

所有敏感权限必须说明用途。

| 权限 | 默认状态 | 用途 |
|---|---|---|
| 麦克风 | 询问开启 | 语音输入 |
| 摄像头 | 默认关闭 | 用户存在和视觉理解 |
| 通知 | 询问开启 | 提醒 |
| 本地存储 | 默认开启 | 设置和记忆 |
| 网络 | 默认开启 | AI Gateway |

### 19.2 摄像头隐私

要求：

- 默认关闭；
- 开启时明确提示；
- 提供明显状态标识；
- 不保存原始视频；
- 本地处理优先；
- 用户可一键关闭。

### 19.3 麦克风隐私

要求：

- 提供按住说话模式；
- 持续监听必须显式开启；
- 可关闭唤醒词；
- 不保存原始录音；
- 语音识别结果可清除。

### 19.4 记忆隐私

要求：

- 记忆必须可查看；
- 记忆必须可删除；
- 用户可关闭记忆；
- 重要记忆写入前可询问用户；
- 不记录敏感信息，除非用户明确要求。

### 19.5 儿童使用注意

如果面向儿童，需要额外设计：

- 家长控制；
- 内容过滤；
- 使用时长限制；
- 不进行情感依赖诱导；
- 不收集儿童敏感数据；
- 不与儿童进行不适当话题。
